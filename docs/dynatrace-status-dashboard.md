# Dynatrace Status Dashboard

The app displays live dependency counts, mapping readiness, Forward path evidence, plan/reconciliation counts, managed
check health, security correlation, and Guardian results.

Status evidence is generated by app functions and Workflow actions. Panels should show network and snapshot identity,
plan digest, profile, create/unchanged/changed/stale/collision counts, mutation counts, and verification state. They must
not show credentials, check names, host inventory, raw API bodies, or detailed path topology.

An empty current telemetry window is missing evidence, never a passing baseline. Seeded and replay data are not a
fallback.

# Forward for Dynatrace v0.12.0

This enterprise preview completes the repository-owned operational controls needed for a design-partner evaluation.
It does not change the single-app architecture or the Forward access-profile contract established in `v0.11.0`.

## Highlights

- Adds named product, support, vulnerability-response, incident, compatibility, and release ownership.
- Defines capability-based Forward and Dynatrace compatibility instead of coupling support to product marketing
  versions.
- Adds a deterministic 100-cycle reconciliation soak over 1,000 relationships to every CI and release gate.
- Documents credential rotation, rate limiting, partial failure, recovery, and longer non-production soak acceptance.
- Adds public support, security reporting, upgrade, and release communication policies.
- Removes environment-specific evidence from the reusable product documentation.
- Preserves immutable plan approval, exact-approved create/update, collision protection, bounded writes, and post-write
  verification.

## Installable

`forward-dynatrace-app-v0.12.0.zip` is the only installable artifact. The SBOM, checksum, signature, and attestation
records are verification evidence. Forward requires no software from this project.

## Verification

Download all release assets together, run `sha256sum -c SHA256SUMS`, and verify the app and SBOM attestations against
`forwardnetworks/forward-dynatrace` before installation.
